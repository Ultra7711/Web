## 2. 랜딩 페이지 목적 및 역할

이 랜딩 페이지는 MixC-world 입점을 전후한 **브랜드 인지 확보 및 첫 인상 형성**을 위한 디지털 명함이자, 샤오홍슈·더우인 SNS 확산의 허브 역할을 한다.

### 목적 3가지

1. **각인** — 브랜드 스토리와 제주 감성을 처음 접하는 중국 소비자에게 각인
2. **전환** — 시그니처 메뉴와 공간 콘셉트를 시각적으로 전달하여 첫 방문/배달 유도
3. **확산** — SNS 링크 연결로 샤오홍슈·더우인으로 트래픽 전환

## 3. 디자인 방향

### 3-1. 비주얼 톤 & 무드

- **키워드**: 밝은 제주 감성, 유채꽃의 화사함, 바다의 청량함, 모던 내추럴 카페
- **레퍼런스**: % Arabica · Blue Bottle · Fuglen (일본 스페셜티) + 제주 자연(유채꽃, 감귤, 바다, 하늘)
- **배경**: 따뜻한 크림 화이트 기반 + 제주 자연 텍스처(유채꽃밭, 해안 블루)로 화사하고 프리미엄한 분위기

## 4. 페이지 구조 (9개 섹션)

### SECTION 01 — Hero (히어로)

**목적**: 첫 인상. 브랜드 세계관 즉시 각인

| 항목 | 내용 |
|------|------|
| 레이아웃 | 100vh 풀스크린, 제주 풍경 풀블리드 배경 이미지 |
| 헤더 | **투명 헤더**가 히어로 이미지 위에 오버레이 (스크롤 시 반투명 배경 전환) |
| 오버레이 | 그라디언트 오버레이 (하단→상단 `rgba(0,0,0,0.4)` ~ `transparent`) |
| 애니메이션 | 로고 페이드인 → 한글 슬라이드업 → 중문 페이드인 → 버튼 등장 (stagger) |

**구조**:
```
┌─────────────────────────────────────────────────┐
│  [로고] YUDONG COFFEE    메뉴 | 스토리 | 매장   │ ← 투명 헤더 (position: fixed)
│─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ──│
│                                                 │
│           ██████████████████████████             │
│           █  제주 풍경 풀블리드 배경  █             │ ← 100vh 풀스크린 이미지
│           ██████████████████████████             │
│                                                 │
│         济州的从容，融入苏州的日常                   │
│         제주의 여유를 쑤저우의 일상 속으로             │
│                                                 │
│         一杯真心，一口美味，一点趣味                  │
│                                                 │
│     [CTA 메뉴 보기 →]    [CTA 매장 안내 →]        │
│                                                 │
│                    ▼ scroll                     │
└─────────────────────────────────────────────────┘
```

**헤더 동작**:
- 초기 상태: `background: transparent`, 히어로 이미지와 자연스럽게 일체감
- 스크롤 후: `background: rgba(250,246,240, 0.95)` + `backdrop-filter: blur(10px)` + 그림자
- 텍스트 색상도 흰색 → 다크 브라운으로 전환

**로고 동작**:
- 호버/클릭 시 버튼 액션 효과 없음 (`cursor: pointer`만 유지, 배경·그림자·스케일 변화 없음)
- 클릭 시 `window.scrollTo({ top: 0, behavior: 'smooth' })` — 화면 최상단으로 부드럽게 이동

**언어 전환 (향후 확장)**:
- 헤더 좌상단 로고 옆에 언어 전환 토글 배치 예정: `中` / `한` / `EN`
- 1차 구현 시에는 미노출 (한·중 병기 고정), i18n 구조만 사전 설계
- 확장 시 `?lang=zh|ko|en` URL 파라미터 또는 토글 클릭으로 전체 페이지 언어 전환

## 6. 기술 스택 제안 (추천: **React + Next.js**)

현재 기획된 요구사항(단순 랜딩)만 보면 HTML/CSS/JS도 훌륭하지만, **향후 Admin 관리자 페이지(CMS 연동)와 다국어(i18n) 확장 계획**을 고려할 때 **React / Next.js** 환경을 강력히 추천합니다.

| 항목 | 선택 | 이유 (Next.js 추천 사유) |
|------|------|--------------------------|
| **프레임워크** | **Next.js (React)** | **SEO 최적화**(SNS 공유 시 유리), **다국어 라우팅**(`?lang=ko` 등) 기본 지원, JSON/API 연동 용이 |
| **스타일링** | Vanilla CSS (또는 CSS Modules) | 별도 프레임워크 락인 없이 순수 CSS로 유연하고 디테일한 디자인 제어 |
| **애니메이션** | Framer Motion (또는 CSS) | 스크롤 트리거, 라우터 전환 등 복잡한 애니메이션을 선언적으로 쉽게 구현 가능 |
| **이미지** | `next/image` 컴포넌트 | WebP 자동 변환, Lazy Loading, 반응형 리사이징 기본 제공 (중국 모바일 환경 최적화) |
| **상태 관리** | React Context API | 메뉴 데이터 연동 및 언어 토글 시 전역 상태 관리 용이 |

**Vanilla HTML/JS 대비 Next.js의 압도적 장점:**
1. **CMS 연동의 용이성**: 추후 메뉴 데이터를 Admin에서 받아올 때(API Fetching 등), React 컴포넌트로 데이터 매핑을 하는 것이 훨씬 깔끔하고 유지보수가 쉽습니다.
2. **다국어(i18n) 확장**: HTML/JS로 다국어를 생코딩하면 DOM 조작이 복잡해지지만, Next.js와 i18n 라이브러리를 쓰면 언어 파일(JSON) 매핑이 매우 간편합니다.
3. **SEO & 공유**: 서버 사이드 렌더링(SSR) 적용으로 위챗, 샤오홍슈 등에서 링크 공유 시 썸네일과 메타데이터가 완벽하게 노출됩니다.

## 7. 다국어 처리 방침

- **기본 언어**: 중국어 간체(简体) 우선
- **병기 방식**: 핵심 문구 한·중 병기 (한국 브랜드 정체성 강조)
- **영문**: 브랜드명 `YUDONG COFFEE` 로마자 고정

**i18n 확장 설계** (한 / 중 / 영 3개 언어):

| 단계 | 구현 내용 |
|------|----------|
| 1차 (현재) | 한·중 병기 고정, 텍스트 HTML 직접 작성. i18n용 `data-i18n` 속성 사전 마킹 |
| 2차 (확장) | `i18n/zh.json`, `i18n/ko.json`, `i18n/en.json` 언어 파일 분리 |
| 3차 (완성) | 헤더 좌상단 언어 토글 UI 노출 + `?lang=` 파라미터 연동 |

```
// i18n 구조 예시 (zh.json)
{
  "hero.tagline": "济州的从容，融入苏州的日常",
  "hero.slogan": "一杯真心，一口美味，一点趣味",
  "menu.sungsandong": "松山洞咖啡",
  ...
}
```
